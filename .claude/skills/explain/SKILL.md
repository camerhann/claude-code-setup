1. **What it does** — one paragraph, plain language, what problem does this solve
2. **Architecture** — how is the code organised? What are the main modules/packages and how do they connect?
3. **Key entry points** — where does execution start? Main files, route handlers, event listeners
4. **Data flow** — how does data move through the system? What are the key models/schemas?
5. **External dependencies** — what services, APIs, databases, or infrastructure does this depend on?
6. **Dev workflow** — how do you build, test, and run this locally? (read package.json, Makefile, docker-compose, etc.)

Read the actual source files to answer — don't guess from filenames alone. Prioritise accuracy over speed.
